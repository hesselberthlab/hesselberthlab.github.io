Class 8 : Python : Basics (2)¶§

Class date:Wednesday 12 February 2014

Goals¶§

  1. Finish concepts from Monday (list slices)

  3. Importing and debugging
  4. Exercises

Review of list concepts¶§

We covered slicing of lists; i.e. retrieving list parts:

In [1]: nums = range(30)

# try this again with implicit start and end
In [2]: nums[5:10]

In [3]: reversed(nums)

# two ways to examine the contents of the iterator,
# same principle for sorted()
In [4]: [i for i in reversed(nums)]

In [5]: list(reversed(nums))

# skip reversed()
In [6]: range(30,0,-1)

# edit a file directly from ipython
In [7]: edit filename.py

Sets¶§

A set is another type in python that lets you store a non-redundant list of items. They support logical operations:

In [8]: skiiers = set(['Tom','Harry','Gurf'])

In [9]: snowboarders = set(['Lucy','Brian','Gurf'])

# intersection
In [10]: skiiers & snowboarders
Out[10]: {'Gurf'}

# union
In [11]: skiiers | snowboarders
Out[11]: {'Brian', 'Gurf', 'Harry', 'Lucy', 'Tom'}

# difference
In [12]: skiiers - snowboarders
Out[12]: {'Harry', 'Tom'}

Equality and Logic¶§

Use if:elif:else statements to test conditions and act on the result. The == and != operators test for equality and inequality, and work on many object comparisons.

# animal colors
In [13]: cat = 'white'

In [14]: dog = 'black'

In [15]: if cat == dog:
   ....:     print "same color"
   ....: elif cat != dog:
   ....:     print "different color"
   ....: else:
   ....:     print "not going to happen"
   ....:

Undefined values in Python¶§

In [16]: this = None

In [17]: bool(this)

In [18]: not this

# the following if statements are equivalent:
In [19]: if this is None:
   ....:     print 'foo'
   ....: 

In [20]: if not this:
   ....:     print 'foo'
   ....: 

# set the following and test with ``not this``
In [21]: this = 0

In [22]: this = ''

Defining functions¶§

You can define functions that encapsulate work flows

In [23]: def square(numlist):
   ....:     result = []
   ....:     for i in numlist:
   ....:         sq = i * i
   ....:         result.append(sq)
   ....:         return result
   ....: 

# or replace loop with: return [i*i for i in numlist]
In [24]: square(nums)

Importing modules¶§

There are a number of modules with objects and functions in the standard library, and there are a also a huge number of Python modules on the web (check github).

To be able to access the contents of a module, you need to import it into your namespace:

In [25]: import math

In [26]: math.log10(1000)
Out[26]: 3.0

In [27]: import sys

Useful python modules¶§

There are several modules in the standard library you will use all the time:

  • sys: sys.argv has all the arguments from the command line
  • collections: espcially defaultdict and Counter
  • itertools: tools for efficient aggregation and iteration
  • argparse: command line option parsing

Debugging Python code¶§

The pdb is the Python Debugger. You can use it to debug programs by dropping you into a shell that allows you to step through the program, line by line.

In [28]: import pdb

# this will drop you into a shell. find the value of ``i`` at the (Pdb)
# prompt
In [29]: for i in range(100):
   ....:     if i == 50:
   ....:         pdb.set_trace()
   ....:

In Class Exercises¶§

  1. Create a list that contains multiple redundant entries. Covert the list to a set with set(list). What happened to the redundant entries?
  2. Open lamina.bed and print the start position of each entry
  3. Print the total coverage of entries in lamina.bed
  4. Convert each row in lamina.bed into a list. Then, print each entry in the list.
  5. Find the average value of entries in lamina.bed (watch out for int / float type issues).
  6. Find the median value of entries in lamina.bed. Then find the mode.
