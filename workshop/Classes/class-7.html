    Class 7 : Python : Basics¶§

    Class date:Monday 10 February 2014

    Goals¶§

    1. Learn the basics of python syntax
    2. Learn to start ipython
    3. Learn basic types in python
    4. Write simple python scripts

    Overview¶§

    Python is a popular programming language that is commonly used for bioinformatics. We will use it to process and filter files. When you can’t write a simple script in awk, it is better to use python.

    The Python documentation [1] is very helpful, with lots of examples. You should read it to become familiar with the language and refer to it when you get stuck.

    [1]Python 2.x docs http://docs.python.org/2/

    Important

    You should begin the Python tutorial [2]. We will cover some language specifics of Python, but will quickly move to using Python in the context of bioinformatic applications.

    [2]Learn Python the Hard Way http://learnpythonthehardway.org/book/

    IPython¶§

    Ipython is an (I)nteractive python terminal that lets you type in python expressions and see the results immediately

    $ ipython

    This command puts you in a shell that accepts python commands, much like the login terminal accepts bash commands.

    Python indentation¶§

    Python depends on proper indentation of your code. This does not work:

    for i in (1, 2, 3):
print i

    Instead you have to indent the print statement to nest it in the for loop:

    for i in (1, 2, 3):
    print i

    Note

    Use spaces instead of tab characters for indentation.

    Update your gedit preferences: Edit -> Preferences, check the Insert spaces instead of tabs box.

    Python Help¶§

    You can find more about any python type or function using pydoc:

    # learn about the python ``string`` type
$ pydoc str

    At the ipython prompt, you can also use:

    In [1]: str?

    In these slides, links will take you to the python docs: str

    Finally, ask Google (e.g. python string split).

    For Loops (iteration)¶§

    Many things in python are iterable, meaning we can write loops over them. For example, a string is iterable:

    In [2]: sentence = 'i LOVE programming'

In [3]: for char in sentence:
   ...:     print char
   ...:

    For Loops (range)¶§

    Automate repetitive tasks with a for loop:

    # Print "hello" 5 times:
In [4]: for i in range(5):
   ...:     print "hello"
   ...: 

# now print the numbers
In [5]: for i in range(5):
   ...:     print i
   ...:

    where the range() function generates the numbers 0, 1, 2, 3, 4.

    Python Types¶§

    There are several core types in Python that you will use a lot.

    • str is a collection of characters (words and sentences).
    • int and float are numbers.
    • list is a group of other objects.
    • dict contains key:value mappings.

    Strings¶§

    A str is a collection of characters. You can make strings with single, double and triple quotes.

    In [6]: phrase = 'this that other'

In [7]: phrase

# uppercase
In [8]: phrase.upper()

# number of characters (including spaces) in phrase
In [9]: len(phrase)

    Numbers (Ints and math)¶§

    Python has integer numbers (int) and floating point numbers (float). Math operations work within and across both types:

    # set up some ints
In [10]: x = 10

In [11]: y = 100

In [12]: type(x)

# add
In [13]: x + y

# subtract
In [14]: x - y

# x * y
In [15]: x * y

    Numbers (Float division)¶§

    For division you need to pay attention to type:

    # try to divide the ints ...
In [16]: x / y

# need float conversion!
In [17]: float(x) / float(y)

# make floats directly and divide
In [18]: x = 10.0

In [19]: y = 100.0

In [20]: type(x)

In [21]: x / y

    Lists¶§

    A list is a collection of other objects. You create lists directly using brackets ([ ]), or they can be created from other objects.

    Lists are subscriptable, meaning that you can access items in a list by position.

    # convert to list, str.split() defaults to space
In [22]: words = phrase.split()

# number of items in list
In [23]: len(words)

# two ways to add new words
In [24]: words.append('foo')

In [25]: words.extend(['bar','baz'])

    # first item only, zero-based
In [26]: words[0]

# first through third, start is implicit
In [27]: words[:3]

# iterate over the list
In [28]: for word in words:
   ....:     print word.capitalize()
   ....: 

# mix types in lists
In [29]: words.extend([1,2,3])

# a "list comprehension"
In [30]: [type(i) for i in words]

    Dictionaries (dicts)¶§

    A dict contains key:value mappings.

    # set up new dicts with {}
In [31]: produce  = {'lettuce':'green', 'apple':'red',
   ....: 'banana':'yellow'}
   ....: 

In [32]: produce.keys()

In [33]: produce.values()

In [34]: produce.items()

# sorted by keys
In [35]: sorted(produce.items())

# test for membership
In [36]: 'apple' in produce

In [37]: not 'orange' in produce

    Python Exceptions¶§

    When you’re learning to program in Python, you will see lots of errors. Examples: ValueError, IndexError and KeyError

    In [38]: int('blah')

In [39]: words[100]

In [40]: parts = {'hip':'thigh'}

In [41]: parts['nose']

# Catch errors, print useful debugging messages
In [42]: try:
   ....:     nums[100]
   ....: except IndexError:
   ....:     print "error: not enough nums"
   ....:

    Reading data from a file¶§

    Now we’ll read some data from a file and operate on each line:

    In [43]: filename = '/opt/bio-workshop/data/lamina.bed'

# What is the BUG in this block?
In [44]: for line in open(filename):
   ....:     fields = line.strip().split('\t')
   ....:     start = fields[1]
   ....:     if start > 5000:
   ....:         print fields
   ....:
